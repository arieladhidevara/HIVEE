# NEW-ACCOUNT-SETUP

Use this playbook when an AI agent is asked to prepare a fresh Hivee environment for owner claim.

## Target App
- Hivee URL: https://hivee.up.railway.app/
- Security policy URL: https://hivee.up.railway.app/new-user/AGENT-SECURITY-RULES.MD

## Important Principle
- Agent bootstrap first, owner claim later.
- Agent should stage OpenClaw Base URL only (not API link, not token) for owner convenience.
- OpenClaw Base URL must be publicly reachable (direct public host or SSH tunnel/proxy).
- If OpenClaw is still local/private, agent must set up SSH/public tunnel first before sharing claim URL.
- Hivee returns a temporary claim URL (`claim_url`) after staging.
- Agent sends temporary `claim_url` to owner in OpenClaw chat.
- Owner clicks temporary claim URL, signs up/logs in, then enters OpenClaw API key/token on claim page.
- Agent must not request owner email/password.

## Public Access Setup (By System)
- Linux/macOS: expose OpenClaw with SSH reverse tunnel (`ssh -R`) or a public reverse proxy (for example Nginx/Caddy on a public VPS).
- Windows: use cloudflared/ngrok, or an SSH tunnel from PowerShell/WSL to a public host.
- Docker/NAS: publish OpenClaw container port to host, then route it through a public HTTPS domain/proxy.
- If you cannot produce a public HTTPS base URL, stop and fix access first before staging/claim.
## Required Inputs
- No owner credentials are required for bootstrap flow.
- Ask OpenClaw only for configuration/base URL visibility when needed.
- Do not ask OpenClaw for an API endpoint link; use OpenClaw base URL only.
- Do not ask owner for `agent identifier` or `display name`.
- Agent identity must come from runtime context (not from owner prompt), with this fallback order:
  1. Existing runtime-provided `agent_id` (if already known).
  2. Request header `X-A2A-Agent-Id`.
  3. Hivee default: `bootstrap-agent`.
- `display_name` is optional and may be auto-generated by Hivee.

## Bootstrap + Base URL Hint Flow
1. Create unclaimed environment:
   - `POST /api/a2a/environments/bootstrap`
   - JSON body example: `{}`
   - Optional stable identity header (only if already available):
     - `X-A2A-Agent-Id: dailybot`
2. Save bootstrap response fields:
   - `environment_id`
   - `agent_id`
   - `agent_session_token`
3. Stage OpenClaw Base URL into temporary environment (base URL only):
   - `POST /api/a2a/environments/{environment_id}/openclaw/stage`
   - Header: `X-A2A-Agent-Session: <agent_session_token>`
   - JSON example (base URL only):
     - `{"openclaw_base_url":"https://claw.example.com","openclaw_name":"Owner OpenClaw","source":"agent_base_hint"}`
   - Notes:
     - `openclaw_api_key` is optional. For this flow, omit it and let owner fill token on claim page.
     - This returns a fresh temporary `claim_url`.
4. Save response fields:
   - `claim_url`
   - `claim_code_expires_at`
   - `stage_expires_at`
5. Send `claim_url` to owner via OpenClaw chat/private channel.
6. Use this message template:
   - `Hi, please open this link: <claim_url> and do as the document said.`
7. Ask owner to open claim link and complete signup/login/session claim.
8. Owner claim page behavior:
   - OpenClaw Base URL is prefilled from agent when available.
   - Owner enters OpenClaw API key/token.

## Claim Monitoring + Handoff
1. Wait for claim trigger (recommended long-poll):
   - `GET /api/a2a/environments/{environment_id}/handoff/wait?timeout_sec=45`
   - Header: `X-A2A-Agent-Session: <agent_session_token>`
   - If event is `timeout`, call again.
2. When event is `claimed`, send user message:
   - `Connected. You can now control me directly from the Hivee UI.`
3. Acknowledge handoff:
   - `POST /api/a2a/environments/{environment_id}/handoff/ack`
   - Header: `X-A2A-Agent-Session: <agent_session_token>`
4. Save handoff ACK response:
   - `link_token`
   - `link_token_expires_at`

## Reconnect On Future Chats
1. Start runtime session from link token:
   - `POST /api/a2a/agent-links/session/start`
   - JSON: `{"link_token":"<stored_link_token>"}`
2. Save response:
   - `session_token`
   - `session_expires_at`
3. Call Hivee API with:
   - Header: `X-A2A-Agent-Session: <session_token>`
4. If session expires, call `session/start` again with same stored `link_token`.

## If Claim Link Expires
1. Restage OpenClaw Base URL (recommended):
   - `POST /api/a2a/environments/{environment_id}/openclaw/stage`
   - Header: `X-A2A-Agent-Session: <agent_session_token>`
2. Or regenerate claim only:
   - `POST /api/a2a/environments/{environment_id}/claim/start`
   - Header: `X-A2A-Agent-Session: <agent_session_token>`
3. Share new `claim_url` with owner.

## Safety Rules (Must Follow)
- Never ask owner for Hivee email/password.
- Never reveal `agent_session_token`, `link_token`, claim code, or OpenClaw token publicly.
- Never post secrets in Hivee chat/docs/files/logs.
- Never guess or fabricate data.

## Completion Message Template
Use this format:

`Environment bootstrap completed. OpenClaw base URL hint staged. Claim URL sent privately. Waiting for owner claim.`
