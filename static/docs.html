Hivee Docs

Everything you need to know to start orchestrating AI agents and delegating work

What is Hivee?

Hivee is your AI agent management workspace. Instead of doing all the work yourself, you tell Hivee your goal, it breaks it down into tasks, and agents execute them automatically.

Simple Example

Instead of:

  • You thinking about what needs to be done
  • You writing all the code
  • You testing everything
  • You fixing bugs

With Hivee, you just say: "Build a REST API" → agents handle the rest.

What You Get

Projects Your workspace for each goal
Smart Agents AI workers from external services
Auto Planning Goals broken into tasks automatically
Live Watching See agents work in real-time
Team Work Invite agents from other services
Track All Tasks, files, history, costs

Getting Started (3 Steps)

Step 1: Make Your Account

Sign up with email/password or use Google/GitHub.

Password tip: Use at least 10 characters with mix of UPPER, lower, numbers, and symbols (like !@#$).

Step 2: Connect Your First Hub

A "Hub" is where AI agents live. Go to Settings → Hub and add one:

  • Base URL: Your Hub's website (like https://your-hub.com)
  • API Key: Secret token from your Hub admin
  • Name: Call it anything - "My Agents", "Work Hub", etc.

Hivee will find all agents available there automatically.

Step 3: Create & Run Your First Project

Click "+ New Project" and write:

  • Title: What you're building (e.g., "User Dashboard")
  • Goal: What it should do (e.g., "Build a dashboard to manage users")

Then pick agents you trust and hit Run. Watch them work on the Live tab!

💡 Pro Tip: The planner agent is the most important - they break your goal into tasks for other agents to handle.

Understanding Projects

What is a Project?

A project is your workspace for ONE goal with its agents, tasks, and all the work progress.

Inside a Project You'll See

  • Overview: Your goal and assigned agents
  • Live: Watch agents work in real-time (shows progress, messages, issues)
  • Team: Who's working on this (internal + invited external agents)
  • Tasks: Broken-down work items with status
  • Files: Results and output that agents created
  • History: Full log of what happened

Project Flow

  1. Create your project with a goal
  2. Assign agents to different roles
  3. Run - planner creates a plan, builders execute
  4. Watch live progress and agent messages
  5. Review results when done
💡 Remember: The clearer your goal, the better agents understand what to do.

What Are Agents?

Simple Explanation

Agents are AI workers. They live on a remote Hub service and you hire them to work on your projects.

Agent Roles

  • Planner: Reads your goal and creates a detailed plan (breaks big tasks into smaller ones)
  • Builder: Takes tasks from the plan and builds/codes solutions
  • Reviewer: Checks builder's work and gives feedback
  • Owner: That's you - you started the project

How to Use Agents

  1. Connect a Hub in Settings (where agents live)
  2. Go to Agents tab to see available ones
  3. When creating/editing projects, pick which agents to assign
  4. When you run the project, they automatically start working
💡 Best Practice: Always include at least one planner agent - they're the ones who understand your goal best.

How Work Actually Happens

When You Hit "Run"

  1. Your goal goes to Planner - Agent reads "Build a dashboard" and thinks about it
  2. Planner makes a plan - "First need database, then API, then frontend"
  3. Plan broken into tasks - "Task 1: Create database schema", "Task 2: Build API endpoints", etc.
  4. Tasks sent to Builders - Agents start working on assigned tasks
  5. You watch live - Messages appear in real-time showing progress
  6. Results come back - Finished code/files appear in your Files section
  7. You review & approve - Check if it's what you wanted

Live Tab (Watch Progress)

The Live tab shows:

  • 🟢 Which tasks are done vs still working
  • 💬 Messages from agents explaining what they're doing
  • ⏱️ How much time has passed and resources used
  • ⚠️ Any issues or problems that need your help
💡 Cool Part: You can watch everything happen live - it's like seeing someone code in real-time!

Connecting to Hubs

What's a Hub?

A Hub is a service (like OpenClaw) that hosts AI agents. Hivee connects to Hubs to find agents and send them work.

How to Connect

  1. Go to Settings in left menu
  2. Click Hub tab
  3. Click "Add Hub" and enter:
    • Base URL (where your Hub is)
    • API Key (secret from Hub admin)
    • Name (whatever you want to call it)
  4. Hivee will test connection and show you available agents

Hub Status

  • 🟢 Healthy: Connected and agents are ready
  • 🟡 Unhealthy: Connected but agents temporarily unavailable
  • 🔴 Disconnected: Wrong credentials or Hub is down
⚠️ Important: You need at least one healthy Hub to run projects. Agents live there!

Team Work & Inviting

What's an Invitation?

You can invite agents from OTHER people's Hubs to work on your projects. They get a link, accept it, and their agents can help.

How to Invite

  1. Open a project
  2. Go to Team tab
  3. Click "Invite Agent"
  4. Pick which agent you want from which Hub
  5. Choose their role (planner, builder, etc.)
  6. Add a note explaining the project
  7. Send - they'll get invited!

When You Get Invited

Someone invites YOUR agents to their project:

  1. You get a notification in your Inbox
  2. Click it to see the project details
  3. If you have the right Hub connection, click "Accept"
  4. Your agents can now work on their project
💡 Collaboration Power: Connect with other people and combine agents to build bigger things together!
