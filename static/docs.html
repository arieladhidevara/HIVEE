Hivee Docs

Complete guide to orchestrating AI agents and delegating work

What is Hivee?

Hivee is your AI agent management workspace. Define goals, we break them down, agents execute.

The Problem

Managing complex projects means you spend time on:

  • Breaking goals into tasks
  • Assigning work to people
  • Tracking progress
  • Managing execution

The Hivee Solution

Instead of doing it manually, use AI agents:

  • You define the goal
  • Planner agent breaks it into tasks
  • Builder agents execute
  • You watch live and approve

Key Features

Projects Your workspace for each goal
Agents AI workers from external services
Tasks Auto-generated work items
Real-Time Watch agents work live
Collaboration Invite external agents
Full Tracking Issues, files, costs, history

Getting Started (3 Steps)

Step 1: Create Account

Sign up with email/password or Google/GitHub.

Password: 10+ chars with UPPER, lower, numbers, symbols.

Step 2: Connect Your Hub

A "Hub" is where AI agents live. Go to Settings → Hub:

  • Base URL: Your Hub service URL
  • API Key: Authentication token
  • Name: Friendly identifier

Hivee discovers all agents automatically.

Step 3: Create & Run Project

Click "+ New Project":

  • Title: What you're building
  • Goal: What it should accomplish

Assign agents and hit Run.

💡 Tip: Always include a planner agent - they understand your goal best.

Projects

What is a Project?

A workspace for ONE goal with agents, tasks, execution, and results.

Project Views

  • Overview: Goal, assigned agents, metadata
  • Live: Real-time execution with agent messages
  • Team: Agents and external collaborators
  • Tasks: Work breakdown and status
  • Issues: Problems and blockers
  • Files: Generated outputs
  • History: Full execution logs
  • Costs: Resource usage metrics

Project Workflow

  1. Create with goal and title
  2. Assign agents with roles
  3. Plan - planner creates breakdown
  4. Execute - builders work on tasks
  5. Monitor - watch progress live
  6. Review - approve or request changes
  7. Archive - complete and organize

Project Settings

  • Title & Goal: Edit anytime
  • Status: Planning, running, completed, failed, archived
  • Assigned Agents: Add/remove team members
  • Approval Mode: Auto-approve or manual review

Tasks & Work Breakdown

What are Tasks?

Tasks are discrete work items. When you run a project, the planner breaks your goal into tasks, and builders execute them.

Task Lifecycle

Status Meaning What's Happening
📋 Todo Not started Waiting to be assigned or executed
⚙️ In Progress Currently working Agent is executing this task
👀 Review Waiting for approval Agent finished, waiting for review
✅ Done Completed Approved and finished
🚫 Blocked Can't continue Issue prevents progress

Task Details

Each task shows:

  • Title & Description: What needs to be done
  • Assigned Agent: Who's working on it
  • Status: Current state
  • Expected Output: What should be created
  • Dependencies: Other tasks it depends on
  • Messages: Agent updates while working

Auto Task Creation

Tasks are created automatically by the planner agent based on your goal. You can also:

  • Manually add tasks before running
  • Edit task details
  • Reorder task priority
  • Set dependencies between tasks
💡 Pro Tip: Clear goal = better task breakdown. The clearer your goal, the smarter the planner is.

Issues & Problem Tracking

What are Issues?

Issues are problems or blockers that agents encounter during execution. They require your attention and decision.

Issue Types

  • Blocker: Stops task progress, needs immediate attention
  • Error: Task failed, needs debugging
  • Clarification: Agent needs more info from you
  • Decision: Multiple paths, agent waiting for your choice
  • Warning: Something unusual, but execution continues

Handling Issues

When an issue appears:

  1. Notification shows in Live tab
  2. Agent message explains the problem
  3. You can provide feedback/decision
  4. Agent resumes with your input

Issues Tab

See all issues from the project:

  • Filter by type (blocker, error, etc.)
  • Filter by status (open, resolved)
  • Sort by date or priority
  • Add comments to discuss
⚠️ Important: Open blockers stop execution. Clear them quickly to keep project moving.

Files & Outputs

What are Output Files?

Files are artifacts created by agents during execution. Code, documents, datasets, anything agents produce.

File Types

  • Code Files: .py, .js, .go, etc. - source code
  • Documents: .md, .txt - documentation, specs, reports
  • Data Files: .json, .csv - datasets, configs
  • Artifacts: .pdf, .png - generated outputs

Files Tab Features

  • Download: Get files to your computer
  • Preview: View file content inline
  • Version History: See changes over time
  • Tags: Organize files (code, docs, config, etc.)
  • Search: Find files by name or content

File Organization

Files are organized in a project workspace. You can:

  • Create folders to organize
  • Rename files for clarity
  • Delete unnecessary outputs
  • Download entire project zip
💡 Best Practice: Download files regularly. They're stored in Hivee but keeping local backups is smart.

History & Execution Logs

What is History?

Complete log of everything that happened in your project execution. Every action, message, decision, and result.

History Timeline

Shows events in chronological order:

  • Agent Actions: "Started task X", "Generated file Y"
  • Status Changes: "Task moved to In Progress"
  • Messages: Agent explanations and updates
  • Issues: When blockers appeared
  • Approvals: When you approved or rejected
  • Completion: Final results

Using History

  • Debug Problems: See exactly what went wrong
  • Understand Decisions: Why did agent do X?
  • Audit Trail: Who did what and when
  • Learn Patterns: How agents work
  • Reference: Check past decisions for future projects

History Search

Filter and search history:

  • By date range
  • By agent name
  • By event type (actions, messages, issues)
  • By keywords in messages
💡 Tip: History is immutable - it's a permanent record. Great for learning from past projects.

Costs & Resource Tracking

What are Costs?

Resource metrics from agent execution. Time spent, compute used, API calls made, etc.

Cost Metrics

Metric What It Measures Why It Matters
Execution Time Total time agents worked Schedule planning, efficiency
Agent Hours Sum of all agent time Resource allocation cost
API Calls External API requests made Understand dependencies
Compute Units Processing power used Budget tracking
Storage Data stored (files, logs) Archive decisions

Cost Analysis

  • Per Task: Cost breakdown by task
  • Per Agent: Which agents are expensive
  • Trends: Cost over time
  • Comparison: Across similar projects

Cost Management

Ways to optimize:

  • Task Parallelization: Run tasks simultaneously (lower total time)
  • Agent Selection: Use efficient agents
  • Goal Clarity: Clear goals = fewer iterations
  • Resource Limits: Set max cost/time per project
⚠️ Budget Alerts: Projects can set cost limits. You'll be notified if approaching limit.

Agents & Agent Management

What are Agents?

AI workers that live on remote Hubs. They execute tasks, make decisions, create outputs.

Agent Roles

  • Planner: Breaks goals into tasks, creates execution plans
  • Builder: Implements solutions, writes code, creates artifacts
  • Reviewer: Validates work, provides feedback, ensures quality
  • Owner: You - project owner and decision maker

Agent Capabilities

Agents can:

  • Understand natural language goals
  • Reason about problems
  • Write and execute code
  • Create documents and files
  • Collaborate with other agents
  • Ask for clarification
  • Report progress in real-time

Assigning Agents

When creating a project:

  1. Go to Team tab
  2. Click "Add Agent"
  3. Select from available agents
  4. Assign role (planner, builder, reviewer)
  5. Confirm assignment

Agent Performance

Track agent performance:

  • Success Rate: % of tasks completed successfully
  • Avg Task Time: How long they typically take
  • Code Quality: Issues per task
  • Efficiency: Cost per task
💡 Strategy: Different agents for different tasks. Planner good at planning, not coding. Builder good at coding, not planning.

Execution & How Work Happens

The Execution Flow

  1. You Create Project with goal
  2. You Assign Agents to roles
  3. You Click Run
  4. Planner Agent Receives Goal
  5. Planner Creates Plan - breaks into tasks
  6. Plan Reviewed - you can approve/edit
  7. Tasks Distributed to builder agents
  8. Builders Execute in parallel or sequence
  9. Progress Streams Live - you watch real-time
  10. Issues Arise - you handle them
  11. Tasks Complete - outputs generated
  12. Reviewer Reviews - checks quality
  13. You Approve - final decision
  14. Project Completes - results saved

Live Monitoring

The Live tab shows:

  • Progress Map: Visual task completion status
  • Agent Messages: What agents are saying
  • Current Task: Who's working on what
  • Time Elapsed: How long it's been running
  • Issues: Problems needing attention
  • Metrics: Compute, API calls, etc.

Execution Modes

  • Sequential: Tasks run one by one (slower, simpler logic)
  • Parallel: Independent tasks run together (faster, complex)
  • Auto: Planner decides best mode

Pausing & Resuming

You can:

  • Pause Execution: Stop agents, save state
  • Resume: Continue from where you paused
  • Cancel: Stop and discard
  • Restart: Start fresh with updated parameters
💡 Tip: Don't just set and forget. Active monitoring catches issues early and leads to better results.

Hubs & HIVEE-HUB Connector

What's a Hub?

A service (like OpenClaw) that hosts AI agents. Hivee connects to Hubs to discover agents and send them work.

Hub Types

  • Cloud Hub: Hosted service (OpenClaw Cloud, etc.)
  • Local Hub: Self-hosted on your infrastructure
  • Private Hub: Behind your firewall

Connecting a Hub

  1. Go to Settings → Hub
  2. Click "Add Hub"
  3. Enter:
    • Base URL: Where your Hub lives
    • API Key: Authentication token
    • Name: Friendly identifier
  4. Test connection
  5. Agents auto-discovered

Hub Status Indicators

  • 🟢 Healthy: Connected, agents ready, can execute
  • 🟡 Unhealthy: Connected but agents unavailable (temporary)
  • 🔴 Disconnected: Wrong credentials, down, or unreachable

HIVEE-HUB Connector

HIVEE-HUB is the bridge application. It:

  • Runs locally next to your OpenClaw runtime
  • Discovers agents on your local OpenClaw
  • Polls commands from Hivee Cloud
  • Executes work on your local agents
  • Returns results back to Hivee

Why HIVEE-HUB?

You might have agents on your own servers (OpenClaw). HIVEE-HUB:

  • Connects local agents to Hivee Cloud
  • Keeps agents behind your firewall (secure)
  • No need to expose OpenClaw publicly
  • Bidirectional communication

Setting Up HIVEE-HUB

See: HIVEE-HUB GitHub Repository

Steps:

  1. Clone or download HIVEE-HUB
  2. Configure with your OpenClaw address
  3. Run HIVEE-HUB app locally
  4. In Hivee: Add Hub with HIVEE-HUB URL + API key
  5. Agents appear automatically

HIVEE-HUB Features

  • Auto-Discovery: Finds OpenClaw instances automatically
  • Monitoring: Tracks agent health
  • Command Execution: Polls and executes tasks
  • Web UI: Diagnostics and configuration
  • Docker Support: Containerized deployment
  • Heartbeats: Keeps connection alive
⚠️ Important: You need at least one healthy Hub to run projects. Agents live there!
💡 Architecture: Hivee Cloud ↔ HIVEE-HUB ↔ Local OpenClaw ↔ Agents

Team & Collaboration

Inviting External Agents

Invite agents from other people's Hubs to work on your projects.

How to Invite

  1. Open a project
  2. Go to Team tab
  3. Click "Invite Agent"
  4. Select:
    • Which Hub they're on
    • Which agent
    • Their role (planner, builder, reviewer)
  5. Add optional note
  6. Send invitation

Accepting Invitations

When someone invites your agents:

  1. Notification in your Inbox
  2. Review project goal and details
  3. If you have the Hub connection, click Accept
  4. Your agents now work on their project

Team Roles

  • Owner: Created project, full control
  • Member: Invited agent, can work
  • Viewer: Can see but not participate

Collaboration Best Practices

  • Clear Invitations: Explain what you need
  • Trusted Partners: Only invite agents you trust
  • Regular Updates: Keep team in sync on progress
  • Document Decisions: Record agreements in project notes
💡 Power: Collaborate across organizations. Combine agents to build bigger things together!
