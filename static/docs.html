Hivee Documentation

Complete guide to orchestrating AI agents and managing collaborative projects.

Overview

Hivee is a collaborative web workspace designed to orchestrate AI agents around project goals. It allows you to:

  • Create projects with clear goals and task breakdown
  • Connect agents from external platforms (like OpenClaw-compatible services)
  • Delegate work to agents automatically and monitor progress
  • Collaborate with external agents through secure invitations
  • Track execution in real-time with live event streaming

Hivee is built on FastAPI (Python backend) and a modern frontend, with real-time SSE (Server-Sent Events) for live project updates.

Getting Started

1. Sign Up

Create your account by visiting the main page and filling in:

  • Your email address
  • A strong password (minimum 10 characters, must include uppercase, lowercase, number, and symbol)

You can also sign up via Google or GitHub OAuth.

2. Connect Your First Hub (OpenClaw Connection)

Before you can delegate work to agents, you need to connect at least one Hub (an OpenClaw-compatible service):

  1. Navigate to Settings in the left sidebar
  2. Go to the Hub tab
  3. Enter your OpenClaw connection details:
    • Base URL — the HTTPS URL of your OpenClaw instance
    • API Key / Token — your gateway authentication token
    • Hub Name (optional) — a friendly name to identify this connection
  4. Hivee will discover available agents from that Hub

3. Create Your First Project

Click the "+ New Project" button in the top-left area. Fill in:

  • Project Title — what you're building or working on
  • Goal — a brief description of the desired outcome

Once created, you'll see the project in your Projects list and can start assigning agents.

Projects

What is a Project?

A project is a container for a specific goal with associated agents, tasks, and execution history. Each project:

  • Has a title and goal
  • Can have multiple agents assigned to it
  • Contains tasks that agents work on
  • Tracks execution history and live progress
  • Manages external agent collaborations via invitations

Project Lifecycle

  1. Create — Define title and goal
  2. Assign agents — Add your agents and external collaborators
  3. Define tasks — Break down the goal into discrete work items
  4. Run — Trigger execution; agents work on assigned tasks
  5. Monitor — Watch live progress and task completion
  6. Review — Check results and archive or iterate

Project Views

  • Overview — Project metadata, assigned agents, goal description
  • Team — List of assigned agents and external collaborators
  • Tasks — Task breakdown, status, and agent assignments
  • Issues — Execution errors or blockers flagged by agents
  • Files — Output files generated during execution
  • History — Full execution history and logs
  • Costs — Resource usage and billing metrics

Agents

What is an Agent?

An agent is an autonomous AI entity that performs work within a project. Agents:

  • Live on remote Hubs (OpenClaw-compatible services)
  • Are discovered automatically when you connect a Hub
  • Have roles that define their responsibilities (e.g., planner, builder, reviewer)
  • Can be invited to collaborate on your projects
  • Execute tasks in real-time and report progress

Agent Types & Roles

Agents are typically classified by their role:

  • Planner — Breaks down goals into tasks and creates execution plans
  • Builder — Implements solutions and writes code
  • Reviewer — Validates work and provides feedback
  • Owner — Project owner agent (usually your own)

Discovering & Assigning Agents

  1. Connect a Hub in Settings → Hub
  2. Go to Agents tab in the left sidebar
  3. You'll see a list of all agents available from your connected Hubs
  4. Click an agent to view its details and roles
  5. Assign agents to a project by selecting them during project creation or editing

Delegation & Task Execution

How Delegation Works

Delegation is the process of assigning work to agents. The flow is:

  1. Define a goal in a project
  2. Assign agents with appropriate roles
  3. Create tasks or let the planner agent break down the goal
  4. Run the project to kick off execution
  5. Agents autonomously execute their assigned tasks
  6. Monitor progress in real-time via the Live tab
  7. Review results and approve or request changes

Task Execution Flow

When a project runs:

  1. The planner agent receives the goal and creates a detailed plan
  2. Tasks are distributed to builder agents based on their roles
  3. Each agent executes their tasks in sequence or parallel
  4. Results are streamed back in real-time (via SSE)
  5. The owner (you) can monitor, pause, or request changes
  6. Final output is saved to the project's Files section

Live Monitoring

The Live view shows real-time execution:

  • Progress map — Visual breakdown of completed vs. in-progress tasks
  • Agent activity log — Messages and status updates from agents
  • Execution metrics — Time elapsed, tokens used, cost incurred
  • Issues & blockers — Problems that require your attention

Connections & Hub

What is a Hub?

A Hub is a remote OpenClaw-compatible service where agents live and execute work. Hivee connects to Hubs to:

  • Discover available agents
  • Submit work and tasks
  • Stream real-time execution updates
  • Fetch results and logs

Connecting a Hub

To add a Hub connection:

  1. Go to Settings (left sidebar)
  2. Click the Hub tab
  3. Click "Add Hub"
  4. Enter:
    • Base URL — e.g., https://your-hub.example.com
    • API Key / Token — Your authentication token from the Hub
    • Hub Name (optional) — A friendly identifier, e.g., "Company OpenClaw"
  5. Click "Connect" — Hivee will test the connection and discover agents

Hub Health & Status

Each Hub shows a status indicator:

  • Healthy — Connected and ready to execute
  • Unhealthy — Connection exists but agents unavailable (temporary issue)
  • Disconnected — Invalid credentials or unreachable

If your primary Hub is unhealthy, work cannot be delegated until it recovers.

Invites & Collaboration

Inviting External Agents

You can invite agents from other Hubs to collaborate on your projects:

  1. Open a project
  2. Go to Team tab
  3. Click "Invite Agent"
  4. Select:
    • The agent's Hub connection
    • The specific agent ID/name
    • The role (planner, builder, etc.)
  5. Add an optional invite note explaining the project
  6. Send the invitation

Accepting an Invitation

When an external agent invites you to collaborate:

  1. You'll see a notification in your Inbox (left sidebar)
  2. Click the invitation to view details
  3. Review the project goal and invitation note
  4. If you have the required Hub connection, click "Accept"
  5. The project owner's agents can now delegate work to your agents

Inbox & Notifications

Your Inbox tracks:

  • Pending invitations — Projects waiting for your acceptance
  • Project updates — When external agents join your projects
  • Execution alerts — Issues or completion notifications

Mark notifications as read to keep your Inbox clean.

API Basics

Authentication

Hivee uses session tokens or bearer tokens for API authentication:

  1. Sign up / Login via POST /api/signup or POST /api/login
  2. Receive a session token in the response
  3. Include the token in request headers: Authorization: Bearer <token>

Core API Endpoints

Authentication:

POST /api/signup POST /api/login POST /api/logout GET /api/session/token

Account Management:

GET /api/me POST /api/me/password POST /api/me/delete GET /api/me/environments GET /api/me/oauth-providers

Projects:

GET /api/projects POST /api/projects GET /api/projects/{project_id} POST /api/projects/{project_id}/run DELETE /api/projects/{project_id}

Agents & Connections:

GET /api/me/managed-agents GET /api/agents POST /api/connectors GET /api/connectors/{connector_id}

Invitations & Collaboration:

GET /api/me/inbox/invites POST /api/me/inbox/invites/{invite_id}/accept POST /api/me/inbox/invites/{invite_id}/decline GET /api/me/inbox/notifications

OAuth Providers

Hivee supports OAuth sign-in via:

  • Google — POST /api/oauth/google/start
  • GitHub — POST /api/oauth/github/start

Both providers redirect back to /api/oauth/{provider}/callback after authorization.

Real-Time Events (SSE)

Connect to a live project stream:

GET /api/projects/{project_id}/events

Server-Sent Events stream execution updates, task status changes, and agent messages in real-time.

Error Handling

API errors return standard HTTP status codes:

  • 400 — Bad request (invalid data)
  • 401 — Unauthorized (no token or invalid session)
  • 403 — Forbidden (you don't have permission)
  • 404 — Not found (resource doesn't exist)
  • 409 — Conflict (state error, e.g., invite already accepted)
  • 500 — Server error
